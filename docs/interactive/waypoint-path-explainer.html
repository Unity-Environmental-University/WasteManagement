The thirty-second précis. Enemies must walk from start to end. The player places road pieces on a grid. This script inspects the pieces, finds the shortest road by the method of Breadth-First Search, and delivers the enemy a list of coordinates to step through, one after the other.

§01
The Problem at Hand

A maze of the player's own making.

The game is, in essence, a maze. The player deposits path pieces on a grid to build a road; enemies must traverse that road to reach the goal at the far end. Simple in concept — thorny in practice.

The player may (and will) do any of the following:

  • Build a road riddled with twists, turns, and branches.
  • Leave dead-ends that lead nowhere in particular.
  • Drop pieces that do not even connect to the main road.

The script must survey this whole mess and determine: what is the best road from start to end?

§04
On the Method

BFS, disassembled one step at a time.

The pond trick. Imagine a pebble dropped into still water. Ripples spread outward in every direction at identical speed. The first ripple to touch the goal marks the shortest route there. This, in a sentence, is breadth-first search.

The three things BFS tracks

  • Frontier — a to-do list of cells to explore next (first in, first out).
  • Came-from map — breadcrumbs recording which cell we arrived from.
  • Goal check — the instant the goal is popped from the frontier, the matter is settled.

Walking the breadcrumbs

BFS leaves behind a came-from map: for every cell it visited, an arrow back toward the cell it arrived from. To reconstruct the path, we start at the goal and follow those arrows home — then reverse the list so the enemy walks start-to-goal.

Each cell remembers one parent. Follow the arrows.
§05
Of Forgiveness in Placement

Anchors search outward in rings.

The start and end points are Transforms dropped into the scene. One need not place them precisely upon a path piece — the script radiates outward in concentric rings until it finds the nearest occupied cell.

Radius caps at 5. Beyond that, the script gives up.

§06
The Code, Disrobed

A tour of the principal routines.

What Rebuild() does

This is the main function. It runs at wave start and performs these actions in order:

  1. Clears the old path.
  2. Adds the start Transform as waypoint #1.
  3. Figures out which grid cell is the BFS start, and which is the goal.
  4. Runs BFS to obtain the cell sequence.
  5. Converts those cells into world positions.
  6. Adds the end Transform as the last waypoint.

If any step fails (no pieces, no connection), it falls back to a straight line from start to end.

The BFS loop, translated to plain English 
// To-do list of cells to check (FIFO queue = shortest path)
var frontier = new Queue<Vector2Int>();
frontier.Enqueue(start);

// Breadcrumbs: for each cell, which cell led us here?
var cameFrom = new Dictionary<Vector2Int, Vector2Int> { [start] = start };

// Only 4 directions — no diagonals!
var directions = new[] {
    new Vector2Int( 1,  0),  // right
    new Vector2Int(-1,  0),  // left
    new Vector2Int( 0,  1),  // up
    new Vector2Int( 0, -1)   // down
};

while (frontier.Count > 0) {
    var current = frontier.Dequeue();

    if (current == goal) break;   // Found it!

    foreach (var dir in directions) {
        var next = current + dir;
        if (cameFrom.ContainsKey(next)) continue;          // already visited
        if (!pathBuildBoard.IsOccupied(next)) continue;    // not a piece
        cameFrom[next] = current;                          // leave breadcrumb
        frontier.Enqueue(next);
    }
}

Why a Queue? A queue is first-in-first-out — the thing you added first comes out first. This makes BFS explore in “layers” of the same distance, which guarantees the shortest path.

Reconstructing the path — the breadcrumb walk 
// Start at goal, walk backward through breadcrumbs
var path = new List<Vector2Int>();
var node = goal;
while (node != start) {
    path.Add(node);
    node = cameFrom[node];   // follow the breadcrumb
}
path.Add(start);
path.Reverse();   // flip to start → goal order

We built the path backwards (goal → start), so we reverse it at the end so the enemy walks in the right direction.

Ring search: ResolveAnchorCell 
// Try the exact cell the Transform is on
if (pathBuildBoard.IsOccupied(anchorCell)) return anchorCell;

// Otherwise, expand outward: ring 1, ring 2, ... up to ring 5
for (var radius = 1; radius <= 5; radius++) {
    for (var dx = -radius; dx <= radius; dx++)
    for (var dy = -radius; dy <= radius; dy++) {
        // Skip the interior — only check the ring's edge
        if (Mathf.Abs(dx) != radius && Mathf.Abs(dy) != radius) continue;

        var candidate = new Vector2Int(anchorCell.x + dx, anchorCell.y + dy);
        if (pathBuildBoard.IsOccupied(candidate)) return candidate;
    }
}
return null;   // nothing found within radius 5

The predicate Mathf.Abs(dx) != radius && Mathf.Abs(dy) != radius skips the interior of the square so we only examine the outer edge of each ring.

What the API exposes to other scripts
  • Count — number of waypoints. Enemies check this to know when they've reached the end.
  • GetPosition(int index) — returns the world position of waypoint index. Enemies call this every frame.
  • Rebuild() — recalculates the path. Call at wave start.
§07
Caveats & Common Troubles

Three ways to bleed.

Diagonal neighbours do not count. BFS checks only up, down, left, right. If two pieces touch only at a corner, the path breaks. Pieces must share a full edge.

Gaps kill the path. A single empty cell between pieces and BFS cannot cross it. The whole road will be marked red (unused) and the enemy walks a straight line from start to end.

Anchors too far from pieces. Ring search caps at radius 5. If the start or end Transform sits more than 5 cells from any placed piece, it fails to resolve.

The scene gizmos

When editing in Unity, you'll see:

  • Yellow lines — the enemy's actual route.
  • Green wire cubes — cells on the path.
  • Red wire cubes — placed pieces that got ignored.

Red cubes are your friend: they tell you which pieces are not in fact helping.

The takeaway. BFS is simply “spread outward until you touch the goal, remember how you got there, then walk the breadcrumbs backward.” That single idea is the entire engine of this script.
